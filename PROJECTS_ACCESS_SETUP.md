# Projects Access Setup (Netlify + Supabase)

This project includes a password-gated Projects section that calls a Netlify Function and redirects valid clients to their project URL.

## 1) Supabase Table

Run this SQL in Supabase:

```sql
create table if not exists public.project_access (
  id bigint generated by default as identity primary key,
  password_hash text not null,
  project_url text not null,
  is_active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_project_access_active on public.project_access (is_active);
```

## 2) Generate Password Hash

Use bcrypt for each client password:

```bash
node -e "const bcrypt=require('bcryptjs'); const pw=process.argv[1]; bcrypt.hash(pw, 12).then(h=>console.log(h));" "CLIENT_PASSWORD_HERE"
```

Copy the generated hash and insert a row in Supabase:

```sql
insert into public.project_access (password_hash, project_url, is_active)
values (
  '$2a$12$REPLACE_WITH_BCRYPT_HASH',
  'https://your-client-project-url.com',
  true
);
```

## 3) Netlify Environment Variables

Set these in Netlify Site Settings > Environment Variables:

- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
- ALLOWED_REDIRECT_HOSTS (comma separated list of allowed hostnames)

Example:

```text
ALLOWED_REDIRECT_HOSTS=client-a.com,client-b.io
```

Only HTTPS redirect targets are allowed.

## 4) Local Testing

1. Install dependencies: `npm install`
2. Build once: `npm run build`
3. Run with Netlify CLI (if installed): `netlify dev`
4. Open the site, go to Projects section, submit a valid password.

## 5) Operational Notes

- One password can map to one project link by default.
- Deactivate access by setting `is_active` to `false`.
- Invalid logins return generic errors and do not reveal whether a project exists.
